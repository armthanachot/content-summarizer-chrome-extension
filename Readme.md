# คู่มืออธิบาย `background.js`

## ไฟล์นี้ใช้ทำอะไร

`background.js` คือสคริปต์ที่ทำงานเบื้องหลังของส่วนขยาย Chrome ตัวนี้ ทำหน้าที่หลัก ๆ ดังนี้

- **จัดการผู้ให้บริการ AI** (เช่น OpenAI กับ Gemini) รวมถึงชื่อโมเดลที่รองรับและค่าเริ่มต้นตามประเภทงาน
- **สร้างเมนูคลิกขวา (Context Menu)** ให้ผู้ใช้สั่งสรุป เปิดแชทด่วน (Fast Chat) หรืออธิบายข้อความที่เลือกบนหน้าเว็บ
- **รับการกดไอคอนส่วนขยาย (Action)** เพื่อเปิด/สลับหน้าต่างทำงานบนหน้าเว็บ
- **รับข้อความจากหน้าเว็บ (content script)** แล้วไปเรียก API ของ AI เพื่อสรุป แปล อธิบายคำ คุยต่อจากบทสรุป หรือดึงหน้าเว็บจาก URL มาสรุป
- **แปลง HTML เป็นข้อความล้วน** เวลาสรุปจากลิงก์ เพื่อตัดโค้ดและส่วนที่ไม่ใช่เนื้อหาหลักออก

สรุปคือ ไฟล์นี้เป็นศูนย์กลางที่เชื่อมระหว่างเมนู ปุ่มส่วนขยาย กับการเรียก AI จริง ๆ

---

## ส่วน Context Menu (เมนูคลิกขวา)

### เมื่อติดตั้งส่วนขยาย (`onInstalled`)

- ลบเมนูเก่าทั้งหมดก่อน แล้วสร้างเมนูใหม่ 3 รายการ เฉพาะตอนที่มีการ **เลือกข้อความ** (`contexts: ['selection']`)
  - **Summarize** — ส่งข้อความที่เลือกไปให้ฝั่งหน้าเว็บเพื่อนำไปสรุปในช่องป้อนข้อมูลของส่วนขยาย
  - **Fast Chat** — แสดงในชื่อเมนูเป็น `💬 Fast Chat` (ไม่ใช้รูป PNG ในเมนูเพราะบางเวอร์ชันของ Chrome ไม่รองรับ `icons` ใน `contextMenus.create`) ส่งข้อความที่เลือกไปฝั่งหน้าเว็บเป็นบริบทสำหรับหน้าต่างแชท ส่วน `icons/assistant.png` ยังใช้ในปุ่มแชทในหน้าต่างส่วนขยายตามเดิม (รายละเอียดด้านล่าง)
  - **Explain in Summary Context** — ส่งข้อความที่เลือกไปเพื่อขอคำอธิบายในบริบทของบทสรุปที่มีอยู่แล้วในหน้าต่างส่วนขยาย

### เมื่อผู้ใช้คลิกเมนู (`onClicked`)

- ตรวจว่ามีแท็บและมีข้อความที่เลือกจริง ถ้าไม่มีก็ไม่ทำอะไร
- ถ้าเลือก **Summarize** จะส่งข้อความไปที่แท็บนั้นด้วยข้อความประเภท `summarize-selection`
- ถ้าเลือก **Fast Chat** จะส่งประเภท `fast-chat-selection` พร้อมข้อความที่เลือก
- ถ้าเลือก **Explain in Summary Context** จะส่งประเภท `explain-selection`
- การส่งข้อความใช้ `sendMessage` ไปที่แท็บ — ถ้าแท็บไม่รับได้ (เช่น หน้าพิเศษ) จะเงียบไว้ ไม่ให้ส่วนขยายพัง

### เมนู Fast Chat ทำงานร่วมกับ `content.js` อย่างไร

แนวคิดสั้น ๆ: **ข้อความที่ไฮไลต์จะถูกมองว่าเป็นบทสรุป (summary) แล้ว** — ใช้เป็นบริบทเดียวกับการกด “คุยกับผู้ช่วย” หลังสรุปจากหน้าต่างหลัก โดยเก็บใน `rawResponse` / `originalResponse` แล้วส่งบริบทนี้ไปที่ `chat-about-summary` ใน `background.js` เมื่อผู้ใช้แชท

**จะไม่เปิดหน้าต่างหลัก (modal ใหญ่)** — เพื่อไม่ให้โค้ด/คีย์บอร์ดของแผงสรุปหลักทำงานร่วม จะแสดง **เฉพาะหน้าต่างแชท** (หัวข้อว่า **Fast Chat**) เท่านั้น

ลำดับการทำงานโดยย่อ:

1. ผู้ใช้เลือกข้อความ แล้วคลิกขวา → **Fast Chat**
2. ฝั่งหน้าเว็บตั้งค่าข้อความที่เลือกให้เทียบเท่าบทสรุปในระบบ แล้วเปิดเฉพาะหน้าต่างแชท (ซ่อน modal หลักไว้)
3. ถ้ามี API key อยู่แล้ว สามารถพิมพ์คุยกับผู้ช่วยได้ทันที
4. ถ้ายังไม่มี API key จะแสดงข้อความแนะนำในหน้าต่างแชทให้ไปตั้งค่าผ่านไอคอนส่วนขยายบนแถบเครื่องมือ แล้วค่อยใช้ Fast Chat อีกครั้ง
5. การกดไอคอนส่วนขยายขณะที่เปิดอยู่แค่ Fast Chat แบบนี้ ครั้งแรกจะ **ปิดหน้าต่างแชท** (ไม่เปิด modal หลัก)

## ส่วน Action (การกดไอคอนส่วนขยาย)

เมื่อผู้ใช้กดไอคอนส่วนขยายบนแถบเครื่องมือ:

1. **ฉีดสคริปต์ `content.js` เข้าแท็บปัจจุบัน** — เพื่อให้มีโค้ดฝั่งหน้าเว็บพร้อมแสดง/ซ่อนหน้าต่างทำงาน
2. **ส่งข้อความ `toggle-modal`** ไปที่แท็บเดียวกัน — เพื่อสลับการเปิดหรือปิดหน้าต่างของส่วนขยายบนหน้าเว็บ

ถ้าแท็บนั้นเป็นหน้าที่ส่วนขยายฉีดสคริปต์หรือส่งข้อความไม่ได้ (เช่น หน้า `chrome://`) จะจับข้อผิดพลาดและแจ้งใน console ว่าทำงานบนหน้านี้ไม่ได้ โดยไม่ให้ผู้ใช้งานสับสนเป็นข้อความเด้งบนหน้าจอ

---

## การรับงานจากหน้าเว็บ (`onMessage`)

นอกจาก Context Menu กับ Action ยังมี **ตัวรับข้อความ** ที่รอคำสั่งจาก `content.js` เช่น สรุป แปล สรุปจาก URL อธิบายคำ และคุยต่อจากบทสรุป แต่ละแบบจะเรียกฟังก์ชันที่เกี่ยวข้องด้านล่าง แล้วตอบกลับว่าสำเร็จหรือมีข้อความผิดพลาด

---

## ฟังก์ชันตั้งแต่ `normalizeProvider` ถึง `extractTextFromHtml`

ด้านล่างอธิบายทีละฟังก์ชันตามลำดับในโค้ด ใช้ภาษาที่เข้าใจได้แม้ไม่ได้ทำงานสายเทคนิค

### `normalizeProvider`

รับชื่อผู้ให้บริการที่ส่งมา (เช่น `openai` หรือ `gemini`) ถ้าชื่อนั้น**มีในรายการที่รองรับ**ก็ใช้ชื่อนั้นต่อไป ถ้า**ไม่รู้จักหรือพิมพ์ผิด** จะเปลี่ยนไปใช้ค่าเริ่มต้น (`openai`) เพื่อไม่ให้โปรแกรมพังเพราะชื่อผิด

### `getProviderLabel`

หลังจากจัดชื่อผู้ให้บริการให้ถูกต้องแล้ว ฟังก์ชันนี้คืนค่า **ชื่อที่อ่านง่าย** สำหรับไปใส่ในข้อความแจ้งเตือน เช่น "OpenAI" หรือ "Gemini"

### `getProviderModel`

เลือก **ชื่อโมเดล** ที่ควรใช้กับงานประเภทหนึ่ง (เช่น สรุป แปล อธิบาย คุย) ตามที่ตั้งไว้ในการตั้งค่า ถ้าไม่มีค่าเฉพาะงานนั้น จะใช้โมเดลเริ่มต้นของผู้ให้บริการนั้น

### `resolveModelSelection`

ตัดสินใจว่าจะใช้ **โมเดลไหน** จริง ๆ: ถ้าผู้ใช้ขอโมเดลมาและโมเดลนั้น**อยู่ในรายการที่รองรับ** ก็ใช้ตามที่ขอ ถ้าไม่ได้ขอหรือขอโมเดลที่ไม่รองรับ จะใช้ค่าจาก `getProviderModel` แทน เพื่อความปลอดภัยและความสอดคล้องกับผู้ให้บริการ

### `initializeAI`

เตรียมข้อมูลก่อนเรียก AI: ตรวจว่ามี **คีย์ API** ไม่ว่าง ถ้าไม่มีจะโยนข้อความผิดพลาดที่บอกชื่อผู้ให้บริการ จากนั้นคืนค่าเป็นก้อนข้อมูลที่มีผู้ให้บริการ คีย์ที่ตัดช่องว่าง และความชอบเรื่องโมเดล

### `generateAIContent`

เป็นจุดที่เลือกว่าจะเรียก **OpenAI** หรือ **Gemini** ตามผู้ให้บริการ โดยเลือกโมเดลจาก `resolveModelSelection` แล้วส่งคำสั่งระบบและคำถามของผู้ใช้ไปยังฟังก์ชันเรียก API ที่เหมาะสม

### `summarizeContent`

สร้างคำสั่งระบบ (system prompt) ให้ AI ทำหน้าที่วิเคราะห์และสรุปเนื้อหาแบบมีโครง มีหัวข้อ จุดสำคัญ และรักษาตัวเลข/ความหมาย ถ้ามี **จำกัดจำนวนคำ** หรือ **ภาษาปลายทาง** จะเติมเงื่อนไขลงในคำสั่ง จากนั้นเรียก AI เพื่อได้บทสรุปกลับมา

### `translateContent`

ให้ AI ทำหน้าที่นักแปลที่เน้นความเป็นธรรมชาติในภาษาปลายทาง รักษารูปแบบ markdown และไม่แก้โค้ด/ลิงก์/ชื่อเฉพาะโดยไม่จำเป็น แล้วส่งเนื้อหาที่ต้องการแปลเข้าไป

### `fetchAndSummarize`

ใช้เมื่อต้องการ **สรุปจากลิงก์**: ดึงหน้า HTML จาก URL (มีการจำกัดเวลาไม่เกิน 30 วินาที) ถ้าดึงไม่ได้หรือ HTTP ไม่สำเร็จจะแจ้งข้อผิดพลาด จากนั้นแปลง HTML เป็นข้อความด้วย `extractTextFromHtml` ถ้าได้ข้อความสั้นเกินไปจะบอกว่าดึงเนื้อหาสาระไม่ได้ (เช่น หน้าว่างหรือต้องใช้ JavaScript แสดงผล) ถ้าผ่านจะเรียก `summarizeContent` กับข้อความที่ได้

### `chatAboutSummary`

ใช้เมื่อผู้ใช้ **คุยต่อจากบทสรุป**: ตรวจว่ามีข้อความบทสรุปและประวัติแชทที่สมบูรณ์ (ข้อความสุดท้ายต้องเป็นฝั่งผู้ใช้และไม่ว่าง) จากนั้นเลือกโมเดลสำหรับงานแชทและเรียก `callOpenAIChat` หรือ `callGeminiChat` ตามผู้ให้บริการ

### `callOpenAIChat`

ส่งบทสรุปเป็นบริบทใน system message รวมกับประวัติการสนทนาไปที่ API แชทของ OpenAI ตั้งอุณหภูมิและขีดจำกัดโทเค็น แล้วคืนข้อความตอบจากผู้ช่วย

### `callGeminiChat`

ทำหน้าที่เดียวกับการคุยกับ Gemini โดยใส่บทสรุปใน **คำสั่งระบบ (system instruction)** และแปลงบทบาทข้อความเป็นรูปแบบที่ Gemini ใช้ (USER/MODEL) แล้วอ่านคำตอบจากโครงสร้างที่ API ส่งกลับ

### `explainWord`

ให้ AI อธิบาย **คำหรือวลี** ที่ผู้ใช้เลือก โดยอิงจากบทสรุปที่ให้มา ใช้ภาษาง่าย ๆ และต้องตอบเป็นภาษาเดียวกับบทสรุป

### `callOpenAI`

เรียก API แบบข้อความเดียว (ไม่ใช่แชทหลายรอบ) ของ OpenAI ส่ง system + user แล้วคืนข้อความคำตอบ หรือข้อความผิดพลาดจาก API

### `callGemini`

เรียก API ของ Gemini โดยจัดวางคำสั่งระบบและคำถามในรูปแบบบทสนทนา MODEL/USER ตามที่ API ต้องการ แล้วรวมข้อความจากส่วนตอบกลับ

### `extractTextFromHtml`

แปลง **HTML ดิบ** ให้เหลือข้อความที่อ่านได้สำหรับนำไปสรุป:

- ตัดแท็กที่มักไม่ใช่เนื้อหาหลัก เช่น สคริปต์ สไตล์ คอมเมนต์ และบล็อกอย่าง nav, footer, header, aside
- แทนที่แท็กปิดบางชนิดด้วยบรรทัดใหม่ แล้วลบแท็กที่เหลือ
- แปลง entity HTML พื้นฐาน (`&nbsp;`, `&amp;` ฯลฯ) และตัวเลขอ้างอิงอักขระ
- รวมช่องว่างซ้ำ ๆ และจำกัดความยาวไม่เกินประมาณ 30,000 ตัวอักษร ถ้าเกินจะตัดและใส่ข้อความบอกว่าเนื้อหาถูกตัด

## สรุปการครอบคลุม

เอกสารนี้อธิบายแล้วว่า `background.js` ทำหน้าที่อะไร การทำงานของ **Context Menu** (รวมเมนู **Fast Chat** และการส่งข้อความ `fast-chat-selection`) และ **Action** รวมถึงการรับข้อความจากหน้าเว็บ และอธิบายฟังก์ชันตั้งแต่ **`normalizeProvider`** จนถึง **`extractTextFromHtml`** ครบทุกฟังก์ชันตามลำดับในไฟล์
