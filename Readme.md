# Content Summarizer

ส่วนขยาย Chrome (Manifest V3) สำหรับสรุป แปล อธิบาย และคุยต่อจากเนื้อหาบนหน้าเว็บ โดยเรียก API ของ **OpenAI** หรือ **Google Gemini** จาก Service Worker ฝั่งพื้นหลัง

| รายการ | ค่าจากโค้ด |
|--------|------------|
| ชื่อ / เวอร์ชัน | `Content Summarizer` · `1.3.0` (`manifest.json`) |
| ไฟล์หลัก | `manifest.json`, `background.js` (service worker), `content.js` (UI บนหน้าเว็บ) |
| สิทธิ์ | `storage`, `activeTab`, `scripting`, `contextMenus` และ `host_permissions`: `<all_urls>` |

## ความสามารถโดยย่อ

- **สรุปข้อความ** — จากช่องป้อนใน modal หรือจากข้อความที่เลือกผ่านเมนูคลิกขวา
- **แปล** — แปลเนื้อหาเป็นเป้าหมายที่เลือก (UI ใน `content.js` รองรับภาษาเช่น ไทย / อังกฤษ)
- **สรุปจาก URL** — ดึง HTML แล้วดึงข้อความก่อนส่งให้ AI (`fetchAndSummarize` ใน `background.js`)
- **คุยกับผู้ช่วยหลังสรุป** และ **Fast Chat** — ใช้ข้อความบทสรุปเป็นบริบท (`chat-about-summary`)
- **แชทแบบ Multimodal** — ในช่องแชทรองรับการวางภาพ (base64/data URL), วางภาพจาก clipboard และแนบไฟล์ภาพผ่านปุ่ม `✚` พร้อม preview ก่อนส่ง
- **อธิบายคำ/วลีในบริบทบทสรุป** — จากเมนูคลิกขวาหรือจาก UI (`explain-word`)

## ผู้ให้บริการ AI และโมเดล (อ้างอิง `PROVIDER_CONFIGS` ใน `background.js`)

- **OpenAI** — โมเดลที่รองรับ: `gpt-4o-mini` (ใช้เป็นค่าเริ่มต้นทุกงาน: สรุป แปล อธิบาย แชท)
- **Gemini** — โมเดลที่รองรับ: `gemini-2.5-flash` (ค่าเริ่มต้นเช่นเดียวกันทุกงาน)

ผู้ใช้เลือกผู้ให้บริการและกรอก API key ใน modal การตั้งค่า; คีย์เก็บใน `chrome.storage.local` ด้วยคีย์ `CONTENT_SUMMARIZER_AI_PROVIDER`, `CONTENT_SUMMARIZER_OPENAI_TOKEN`, `CONTENT_SUMMARIZER_GEMINI_TOKEN` (ดู `STORAGE_KEYS` ใน `content.js`)

## ข้อความจาก `content.js` → `background.js` (`chrome.runtime.onMessage`)

| `type` | หน้าที่ |
|--------|---------|
| `summarize` | สรุปเนื้อหา (`summarizeContent`) — รองรับ `maxWords`, `targetLang`, `sameLanguageAsContent`, `model` |
| `translate` | แปล (`translateContent`) |
| `summarize-url` | ดึง URL แล้วสรุป (`fetchAndSummarize`) |
| `explain-word` | อธิบายคำในบริบทบทสรุป (`explainWord`) |
| `chat-about-summary` | แชทต่อจากบทสรุป (`chatAboutSummary`) — ส่ง `summaryContext` และ `messages` |

# คู่มืออธิบาย `background.js`

## ไฟล์นี้ใช้ทำอะไร

`background.js` คือ service worker ที่ทำงานเบื้องหลังของส่วนขยาย ทำหน้าที่หลัก ๆ ดังนี้

- **จัดการผู้ให้บริการ AI** (OpenAI กับ Gemini) รวมถึงรายการโมเดลที่รองรับและค่าเริ่มต้นตามประเภทงาน (`PROVIDER_CONFIGS`)
- **สร้างเมนูคลิกขวา (Context Menu)** ให้ผู้ใช้สรุป เปิด Fast Chat หรืออธิบายข้อความที่เลือกบนหน้าเว็บ
- **รับการกดไอคอนส่วนขยาย (Action)** เพื่อฉีด `content.js` (ถ้าจำเป็น) แล้วส่ง `toggle-modal` เพื่อเปิด/สลับหน้าต่างทำงานบนหน้าเว็บ
- **รับข้อความจาก content script** แล้วไปเรียก API ของ AI เพื่อสรุป แปล อธิบายคำ คุยต่อจากบทสรุป หรือดึงหน้าเว็บจาก URL มาสรุป
- **แปลง HTML เป็นข้อความล้วน** เวลาสรุปจากลิงก์ เพื่อตัดโค้ดและส่วนที่ไม่ใช่เนื้อหาหลักออก (`extractTextFromHtml`)

สรุปคือ ไฟล์นี้เป็นศูนย์กลางที่เชื่อมระหว่างเมนู ปุ่มส่วนขยาย กับการเรียก AI จริง ๆ

## ส่วน Context Menu (เมนูคลิกขวา)

### เมื่อติดตั้งส่วนขยาย (`onInstalled`)

- ลบเมนูเก่าทั้งหมดก่อน แล้วสร้างเมนูใหม่ 3 รายการ เฉพาะตอนที่มีการ **เลือกข้อความ** (`contexts: ['selection']`)
  - **`cs-summarize-selection`** — ชื่อเมนู `🔄 Summarize` — ส่งข้อความที่เลือกไปให้ฝั่งหน้าเว็บเพื่อเปิด modal พร้อมข้อความนั้น
  - **`cs-fast-chat`** — ชื่อเมนู `💬 Fast Chat` (ไม่ใช้รูป PNG ในเมนูเพราะบางเวอร์ชันของ Chrome ไม่รองรับ `icons` ใน `contextMenus.create`) — ส่งข้อความที่เลือกไปฝั่งหน้าเว็บสำหรับ Fast Chat; ไอคอน `icons/assistant.png` ยังใช้ในปุ่มแชทใน UI ของส่วนขยายตามเดิม
  - **`cs-explain-word`** — ชื่อเมนู `🔍 Explain in Summary Context` — ส่งข้อความที่เลือกไปเพื่อขอคำอธิบายในบริบทของบทสรุปที่มีอยู่แล้วในหน้าต่างส่วนขยาย (ต้องมีบทสรุปและ API key อยู่แล้ว ฝั่ง `content.js` จะเรียก `explain-word`)

### เมื่อผู้ใช้คลิกเมนู (`onClicked`)

- ตรวจว่ามีแท็บและมีข้อความที่เลือกจริง ถ้าไม่มีก็ไม่ทำอะไร
- ถ้าเลือก **Summarize** จะส่งข้อความไปที่แท็บนั้นด้วยข้อความประเภท `summarize-selection` พร้อม `term`
- ถ้าเลือก **Fast Chat** จะส่งประเภท `fast-chat-selection` พร้อม `term`
- ถ้าเลือก **Explain in Summary Context** จะส่งประเภท `explain-selection` พร้อม `term`
- การส่งข้อความใช้ `chrome.tabs.sendMessage` — ถ้าแท็บไม่รับได้ (เช่น หน้าพิเศษ) จะเงียบไว้ ไม่ให้ส่วนขยายพัง

### เมนู Fast Chat ทำงานร่วมกับ `content.js` อย่างไร

แนวคิดสั้น ๆ: **ข้อความที่ไฮไลต์จะถูกมองว่าเป็นบทสรุป (summary)** — ตั้ง `rawResponse` / `originalResponse` แล้วเปิดเฉพาะแผงแชท (หัวข้อ **Fast Chat**) โดยซ่อน modal หลัก (`openStandaloneFastChat`)

**จะไม่เปิดหน้าต่างหลัก (modal ใหญ่)** — แสดงเฉพาะหน้าต่างแชทเท่านั้น

ลำดับการทำงานโดยย่อ:

1. ผู้ใช้เลือกข้อความ แล้วคลิกขวา → **Fast Chat**
2. ฝั่งหน้าเว็บตั้งค่าข้อความที่เลือกให้เทียบเท่าบทสรุป แล้วเปิดเฉพาะแผงแชท (ซ่อน modal หลัก)
3. ถ้ามี API key อยู่แล้ว สามารถพิมพ์คุยกับผู้ช่วยได้ทันที (ส่ง `chat-about-summary` ไปที่ `background.js`)
4. ถ้ายังไม่มี API key จะแสดงข้อความแนะนำในแผงแชทให้ไปตั้งค่าผ่านไอคอนส่วนขยาย
5. การกดไอคอนส่วนขยายขณะที่เปิดอยู่แค่ Fast Chat แบบนี้: ใน `toggleModal` ถ้าแผงแชท Fast Chat โหมด standalone แสดงอยู่ ครั้งแรกจะ **ปิดแผงแชท** และรีเซสชัน (ไม่เปิด modal หลัก)

## ส่วน Action (การกดไอคอนส่วนขยาย)

เมื่อผู้ใช้กดไอคอนส่วนขยายบนแถบเครื่องมือ:

1. **พยายาม executeScript ฉีด `content.js` เข้าแท็บปัจจุบัน** — กรณีที่ยังไม่มีสคริปต์ (เช่น หน้าเปิดก่อนติดตั้งส่วนขยาย)
2. **ส่งข้อความ `toggle-modal`** ไปที่แท็บเดียวกัน — เพื่อสลับการเปิดหรือปิดหน้าต่างของส่วนขยายบนหน้าเว็บ

ถ้าแท็บนั้นเป็นหน้าที่ส่วนขยายฉีดสคริปต์หรือส่งข้อความไม่ได้ (เช่น หน้า `chrome://`) จะจับข้อผิดพลาดและแจ้งใน console ว่าทำงานบนหน้านี้ไม่ได้ โดยไม่ให้ผู้ใช้งานสับสนเป็นข้อความเด้งบนหน้าจอ

## การรับงานจากหน้าเว็บ (`onMessage`)

นอกจาก Context Menu กับ Action ยังมี **ตัวรับข้อความ** ที่รอคำสั่งจาก `content.js` ตามตารางด้านบน แต่ละแบบจะเรียกฟังก์ชันที่เกี่ยวข้องด้านล่าง แล้วตอบกลับ `{ success, data }` หรือ `{ success: false, error }`

## ฟังก์ชันตั้งแต่ `normalizeProvider` ถึง `extractTextFromHtml`

ด้านล่างอธิบายทีละฟังก์ชันตามลำดับในโค้ด ใช้ภาษาที่เข้าใจได้แม้ไม่ได้ทำงานสายเทคนิค

### `normalizeProvider`

รับชื่อผู้ให้บริการที่ส่งมา (เช่น `openai` หรือ `gemini`) ถ้าชื่อนั้น**มีใน `PROVIDER_CONFIGS`** ก็ใช้ชื่อนั้นต่อไป ถ้า**ไม่รู้จักหรือพิมพ์ผิด** จะเปลี่ยนไปใช้ค่าเริ่มต้น (`openai`)

### `getProviderLabel`

หลังจากจัดชื่อผู้ให้บริการให้ถูกต้องแล้ว ฟังก์ชันนี้คืนค่า **ชื่อที่อ่านง่าย** สำหรับไปใส่ในข้อความแจ้งเตือน เช่น "OpenAI" หรือ "Gemini"

### `getProviderModel`

เลือก **ชื่อโมเดล** ที่ควรใช้กับงานประเภทหนึ่ง (`summarize`, `translate`, `explain`, `chat`) ตาม `taskDefaults` ใน `PROVIDER_CONFIGS` ถ้าไม่มีค่าเฉพาะงานนั้น จะใช้ `models.default`

### `resolveModelSelection`

ตัดสินใจว่าจะใช้ **โมเดลไหน** จริง ๆ: ใช้ `client.modelPreference` ถ้าเป็น string และอยู่ใน `models.supported` ของผู้ให้บริการนั้น มิฉะนั้นใช้ค่าจาก `getProviderModel`

### `initializeAI`

เตรียมข้อมูลก่อนเรียก AI: ตรวจว่ามี **คีย์ API** ไม่ว่าง ถ้าไม่มีจะโยนข้อความผิดพลาดที่บอกชื่อผู้ให้บริการ จากนั้นคืนค่าเป็นก้อนข้อมูลที่มีผู้ให้บริการ คีย์ที่ตัดช่องว่าง และ `modelPreference`

### `generateAIContent`

เป็นจุดที่เลือกว่าจะเรียก **OpenAI** หรือ **Gemini** ตามผู้ให้บริการ โดยเลือกโมเดลจาก `resolveModelSelection` แล้วส่งคำสั่งระบบและคำถามของผู้ใช้ไปยัง `callOpenAI` หรือ `callGemini`

### `summarizeContent`

สร้างคำสั่งระบบ (system prompt) ให้ AI ทำหน้าที่วิเคราะห์และสรุปเนื้อหาแบบมีโครง มีหัวข้อ จุดสำคัญ และรักษาตัวเลข/ความหมาย ถ้ามี **จำกัดจำนวนคำ** (`maxWords`) หรือ **ภาษาปลายทาง** (`targetLang`) จะเติมเงื่อนไขลงในคำสั่ง ถ้าไม่ระบุภาษาแต่ตั้ง `sameLanguageAsContent` จะขอให้สรุปเป็นภาษาเดียวกับเนื้อหาต้นทาง จากนั้นเรียก AI เพื่อได้บทสรุปกลับมา

### `translateContent`

ให้ AI ทำหน้าที่นักแปลที่เน้นความเป็นธรรมชาติในภาษาปลายทาง รักษารูปแบบ markdown และไม่แก้โค้ด/ลิงก์/ชื่อเฉพาะโดยไม่จำเป็น แล้วส่งเนื้อหาที่ต้องการแปลเข้าไป

### `fetchAndSummarize`

ใช้เมื่อต้องการ **สรุปจากลิงก์**: ดึงหน้า HTML จาก URL (มี `AbortController` จำกัดเวลาไม่เกิน 30 วินาที) ถ้าดึงไม่ได้หรือ HTTP ไม่สำเร็จจะแจ้งข้อผิดพลาด จากนั้นแปลง HTML เป็นข้อความด้วย `extractTextFromHtml` ถ้าได้ข้อความสั้นเกินไป (น้อยกว่า 20 ตัวอักษรหลัง trim) จะบอกว่าดึงเนื้อหาสาระไม่ได้ ถ้าผ่านจะเรียก `summarizeContent` กับข้อความที่ได้

### `chatAboutSummary`

ใช้เมื่อผู้ใช้ **คุยต่อจากบทสรุป**: ใช้ค่าคงที่ `CHAT_SYSTEM_INSTRUCTION` ร่วมกับบทสรุปใน system / systemInstruction ตรวจว่ามี `summaryContext` และประวัติแชทที่สมบูรณ์ (ข้อความสุดท้ายต้องเป็นฝั่งผู้ใช้และไม่ว่าง) จากนั้นเลือกโมเดลสำหรับงาน `chat` และเรียก `callOpenAIChat` หรือ `callGeminiChat` ตามผู้ให้บริการ

### `callOpenAIChat`

ส่งบทสรุปเป็นบริบทรวมกับประวัติการสนทนาไปที่ OpenAI Responses API (`/v1/responses`) โดยฝั่งผู้ใช้ส่งเป็น multimodal input (ข้อความ + รูปภาพที่แนบ) แล้วอ่านข้อความตอบจาก `output[].content[].text`

### `callGeminiChat`

ทำหน้าที่เดียวกับการคุยกับ Gemini โดยใส่บทสรุปใน **systemInstruction** และแปลงบทบาทข้อความเป็นรูปแบบ USER/MODEL แล้วอ่านคำตอบจาก `candidates[0].content.parts`

### `explainWord`

ให้ AI อธิบาย **คำหรือวลี** ที่ผู้ใช้เลือก โดยอิงจากบทสรุปที่ให้มา (`context`) ใช้ภาษาง่าย ๆ และต้องตอบเป็นภาษาเดียวกับบทสรุป

### `callOpenAI`

เรียก API แบบข้อความเดียว (ไม่ใช่แชทหลายรอบ) ของ OpenAI ส่ง system + user แล้วคืนข้อความคำตอบ หรือข้อความผิดพลาดจาก API

### `callGemini`

เรียก `generativelanguage.googleapis.com/v1/models/{model}:generateContent` โดยจัดวาง system prompt และ user prompt ใน `contents` เป็น MODEL แล้ว USER ตามที่ API ต้องการ (กรณีแชทผู้ใช้รองรับ parts แบบ text + inline_data) แล้วรวมข้อความจากส่วนตอบกลับ

### `extractTextFromHtml`

แปลง **HTML ดิบ** ให้เหลือข้อความที่อ่านได้สำหรับนำไปสรุป:

- ตัดแท็กที่มักไม่ใช่เนื้อหาหลัก เช่น สคริปต์ สไตล์ คอมเมนต์ และบล็อกอย่าง nav, footer, header, aside
- แทนที่แท็กปิดบางชนิดด้วยบรรทัดใหม่ แล้วลบแท็กที่เหลือ
- แปลง entity HTML พื้นฐาน (`&nbsp;`, `&amp;` ฯลฯ) และตัวเลขอ้างอิงอักขระ
- รวมช่องว่างซ้ำ ๆ และจำกัดความยาวไม่เกิน `MAX_CHARS` (30,000 ตัวอักษร) ถ้าเกินจะตัดและใส่ข้อความ `[Content truncated...]`

## สรุปการครอบคลุม

เอกสารนี้อธิบายภาพรวมโปรเจกต์ **Content Summarizer** และว่า `background.js` ทำหน้าที่อะไร การทำงานของ **Context Menu** (รวมเมนู **Fast Chat** และการส่งข้อความ `fast-chat-selection`) และ **Action** รวมถึงการรับข้อความจากหน้าเว็บ และอธิบายฟังก์ชันตั้งแต่ **`normalizeProvider`** จนถึง **`extractTextFromHtml`** ครบตามลำดับในไฟล์
